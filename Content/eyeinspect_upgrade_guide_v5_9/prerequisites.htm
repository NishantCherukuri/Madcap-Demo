Prerequisites

This page describes the prerequisites required before upgrading to eyeInspect v5.9.

Prepare to Upgrade

  • Upgrade method: All eyeInspect components should be upgraded via SSH.
  • Upgrade order: When upgrading eyeInspect, begin with the Command Center, followed by the Passive Sensors, and finally the Active Sensors.
  • Compatibility: When upgrading eyeInspect, the upgraded eyeInspect Command Center is considered compatible if a Passive Sensor can connect and provide data to it.
  • Backup: It is recommended to perform a backup of each eyeInspect component before the upgrade. Please refer to the Configuration Guide for backup and restore procedures.

Prepare the Necessary Files

The following files must be downloaded from the Forescout Support Portal before starting the upgrade:

File Version Description
eyeinspect_command-center_5.9.1_update.run 5.9.1 This script upgrades the Command Center
eyeinspect_sensor_5.9.1_update_amd64.run 5.9.1 This script upgrades the Passive Sensor
eyeinspect_active-sensor_2.2.1_update_amd64.run 2.2.1 This script upgrades the Active Sensor

Special Note About FS-HW-2130

The installation/upgrade steps for FS-HW-2130 are the same as for eyeInspect, with one exception: while running the deployers, it is necessary to have a free space of minimum 8 GB.

To check available free space, run the following command: 

$ FREE_ROOT_KB=$(df --block-size 1024 | grep "/$" | awk '{print $4}');FREE_ROOT_GB=$((FREE_ROOT_KB / 1024 / 1024));echo ${FREE_ROOT_GB}GB

If the hardware doesn't meet the minimum required free space criteria, it is recommended to free up as much space as possible, to allow the deployer to run successfully.

Navigate to the home directory and delete files and folders in the /home/silentdefense folder (except the deployer file), by running the following commands:

$ cd /home/silentdefense
$ sudo rm <filename> # delete file
$ sudo rm -rf <foldername> # deletes folder along with all its contents

This will free up some space for install/upgrade.

Warning: Ensure that the deployer to be installed is not deleted.

You can also run the install/upgrade deployers by skipping the readiness check. This ensures that the validation for minimum free disk space (8GB) that is done while running the deployer, is skipped. Run the following command to do so: 

$ sudo ./<runfile> -- --skip-readinesscheck

Upon seeing the following message:

Are you sure you want to continue? [y/N],

Press y and press Enter.
